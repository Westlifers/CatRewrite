# CatRewrite Progress Report 2

## Current Status

CatRewrite has advanced from a textual proof prototype plus early diagram interaction into a more coherent diagrammatic proof workflow.

The main architectural invariant still holds:

```text
Vue is for interaction.
TypeScript core is for mathematics.
Lean is optional verification later.
```

The current project remains an npm workspace:

```text
packages/core
packages/app
```

Core proof logic is still in `packages/core`; Vue components call core APIs and do not implement the proof kernel.

## Major Work Completed Since `progress/1.md`

### 1. Diagram Proof State

Added a formal diagram proof-state layer in `packages/core/src/diagramProof.ts`.

Implemented:

- `DiagramProofState`;
- `DiagramSubgoal`;
- `createDiagramProofState`;
- `createSubgoalFromPaths`;
- `createSubgoalFromRegion`;
- `splitDiagramGoal`;
- `splittableRegions`;
- `proveDiagramSubgoal`;
- `diagramSubgoalRules`.

This means selected diagram paths and declared diagram regions can now become real proof-state subgoals, not just a visual selection.

Subgoals store:

- id;
- region id;
- label;
- left/right paths;
- original equation;
- current equation;
- status;
- proof steps.

### 2. Proved Subgoals as Local Rewrite Rules

Implemented local rewrite rule generation from proved diagram subgoals.

Once a subgoal is proved, it contributes rules through:

```ts
diagramSubgoalRules(state)
```

Rules are tagged:

```text
simp
user
```

A direct rule rewrites the proved left path to the proved right path.

Also added a tail variant:

```text
proved_path >> tail  ->  replacement_path >> tail
```

This is important for pasting: a proved square can rewrite the beginning of a longer outer path.

### 3. Complete Goal by Subgoals

The app now has an explicit proof workflow:

1. split the diagram into subgoals;
2. prove each subgoal;
3. click `Complete Goal`;
4. the main goal is closed using proved subgoals as local rules.

This is deliberately explicit rather than fully automatic. It matches the current maturity of the system better than automatic region inference.

### 4. Identity Morphism Diagram Fix

Fixed the awkward identity/self-loop rendering issue in equation-generated diagrams.

When a goal has the same source and target object, the diagram generator now creates two endpoint nodes carrying the same formal object:

```text
X ----id(X)----> X
```

instead of drawing a zero-length/self arrow.

This is implemented in `equationDiagram`.

Covered by a test for:

```text
id(X) = id(X)
```

### 5. General Natural Transformations

Added first-class natural transformation support.

Implemented in core:

- `NatTransExpr`;
- `NatTransDecl`;
- component terms:

```text
alpha_X
alpha_Y
```

- parser support:

```text
nattrans alpha : F => G
```

- typechecking for components:

```text
alpha_X : F X -> G X
```

- pretty-printing for components;
- equality for natural transformations and components;
- rewrite pattern support for components;
- generated naturality rules:

```text
F.map(f) >> alpha_Y = alpha_X >> G.map(f)
```

- tactic support:

```text
naturality alpha at f
```

### 6. First True Diagram-Splitting Example

Added a nontrivial example where splitting is genuinely needed:

```text
vertical-composite-naturality
```

Mathematical situation:

```text
F, G, H : C -> D
alpha : F => G
beta : G => H
f : X -> Y
```

Main goal:

```text
F.map(f) >> alpha_Y >> beta_Y
=
alpha_X >> beta_X >> H.map(f)
```

The diagram has three rows:

```text
F X ----F.map(f)----> F Y
 |                    |
alpha_X             alpha_Y
 |                    |
 v                    v
G X ----G.map(f)----> G Y
 |                    |
beta_X              beta_Y
 |                    |
 v                    v
H X ----H.map(f)----> H Y
```

The outer rectangle is split into:

```text
F.map(f) >> alpha_Y = alpha_X >> G.map(f)
```

and:

```text
G.map(f) >> beta_Y = beta_X >> H.map(f)
```

These prove by:

```text
naturality alpha at f
naturality beta at f
```

Then the main goal closes by pasting via local rewrite rules.

## Important Current Design Decisions

### Manual / Guided Splitting Before Automatic Splitting

The current system supports:

- explicit path selection;
- declared diagram regions;
- guided splitting via `Split Goal`;
- manual subgoal creation via `Add Subgoal`.

It does not yet infer arbitrary regions automatically.

This is intentional. Automatic region inference should come later, once the proof-state workflow is stable.

### Subgoals Are Proof-State Objects

Subgoals conceptually belong with the main goal, not with the diagram controls.

The diagram helps create and visualize them, but proof state lives in the center proof column.

### Proved Subgoals Are Local Rules

Pasting is currently implemented by converting proved subgoals into local rewrite rules and running the main goal through the tactic engine.

This is formal and useful, but still a first version of pasting.

## Current Limitations

### 1. Pasting Is Rewrite-Based, Not Yet a Dedicated Proof Step

`Complete Goal` works by running `try` with local proved-subgoal rules.

The proof trace says that `try` closed the goal, but it does not yet produce a human-readable pasting trace like:

```text
by alpha-square
by beta-square
therefore outer by pasting
```

### 2. Declared Regions Are Hardcoded for the Split Example

The vertical composite naturality diagram declares `alpha-square` and `beta-square`.

There is no general algorithm yet to infer squares, triangles, or rectangles from arbitrary diagrams.

### 3. Auxiliary Arrow UI Is Still Text-Based

Auxiliary arrows are formally checked, but source/target are still entered as text fields.

The intended later interaction remains:

```text
click source node
click target node
enter term
add verified arrow
```

### 4. Natural Transformation Support Is Basic

General natural transformations now exist, but support is still minimal:

- no vertical composite term constructor;
- no explicit natural transformation composition declaration;
- no naming of composite natural transformations;
- no automatic recognition that `beta after alpha` is a natural transformation.

The current example proves the componentwise equation directly.

### 5. UI Still Needs Real Interaction Testing

Builds pass, but no browser-level UI tests exist yet.

The Browser plugin skill was available in previous work, but the callable browser-control tool was not exposed in this session.

## Recommended Next Steps

### 1. Improve Pasting Trace

Add a dedicated completion step rather than only using `try`.

Suggested API:

```ts
completeDiagramGoalBySubgoals(ctx, diagramProofState, proofState)
```

It should:

- require all subgoals proved;
- apply local rules to the outer goal;
- mark the outer goal proved;
- produce a proof step with a message like:

```text
completed by pasting alpha-square and beta-square
```

This would make the proof trace much more mathematically meaningful.

### 2. Mark the Outer Region Complete

When `Complete Goal` proves the main goal, mark the outer region as proved too.

Current visual marking only marks proved subgoal regions.

Expected later behavior:

```text
alpha-square OK
beta-square OK
outer OK
```

### 3. Add Better Region Controls

The right diagram panel should eventually show declared regions:

```text
outer
alpha-square
beta-square
```

and let the user select a region directly.

For now `Split Goal` is enough, but region visibility will matter for more complex examples.

### 4. Improve Subgoal Panel Ergonomics

Possible improvements:

- show the main goal as the first item in the proof-state panel;
- show subgoal dependency / parent goal;
- show proof tactic used per subgoal;
- auto-select the next open subgoal after proving one;
- expose a small status summary:

```text
2/2 subgoals proved
```

### 5. Node-Based Auxiliary Arrow UI

Implement direct node selection for auxiliary arrows:

1. click `Add Auxiliary`;
2. click source node;
3. click target node;
4. enter term;
5. add verified arrow.

Keep text fields as fallback/debug mode.

### 6. Better Diagram Layout

The vertical composite example is hand-laid out.

Future layout should handle:

- declared rectangular grids;
- shared intermediate rows;
- identity arrows via duplicate endpoints;
- long paths;
- proved regions;
- auxiliary arrows.

### 7. LaTeX Export After Pasting Trace

Once pasting has a real proof trace, add LaTeX export.

The user prefers `CD`, not `tikzcd`.

Export targets:

- context summary;
- aligned proof calculation;
- CD diagram;
- proof by subdiagram pasting.

### 8. Lean Export Later

Lean export remains later.

Current priority should stay on:

```text
diagrammatic proof state -> subgoals -> tactics -> pasting trace
```
